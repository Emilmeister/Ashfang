# MVP scope и критерии успеха (DEV-8)

## 1) Цель MVP
Собрать и выпустить **играбельный браузерный vertical slice** `Ashfang: Echoes of the Fallen Wild`, который демонстрирует core-loop: перемещение -> бой -> прохождение короткого уровня -> базовый результат сессии.

## 2) Ключевые сценарии игрока (3-5)
1. **Первый запуск и старт игры:** игрок открывает игру в браузере, начинает новую сессию без внешней помощи.
2. **Освоение управления:** игрок использует WASD/стрелки, понимает цель уровня через onboarding.
3. **Бой и выживание:** игрок вступает в бой минимум с одним типом врага, получает/наносит урон, завершает encounter.
4. **Прохождение демо-уровня:** игрок доходит до финиша и получает экран результата (win/lose).
5. **Повторная сессия:** игрок возвращается в игру после выхода, базовый прогресс/состояние восстанавливается (MVP-save).

## 3) In scope (MVP)
- Один играбельный уровень длительностью ~5-10 минут.
- Один персонаж игрока с базовым набором действий: движение, одна атака, получение урона.
- Один тип врага с простым поведением.
- Базовые коллизии и физика (стабильный gameplay loop).
- Базовый HUD (HP/статус) и меню паузы.
- Минимальный onboarding в самом уровне.
- Базовые ассеты (допустимы временные плейсхолдеры при отсутствии блокеров).
- Локальное сохранение минимального прогресса и настроек.
- Событийная аналитика для критических точек воронки.
- CI quality gate + сборка и deployment checklist перед релизом.

## 4) Out of scope (после MVP)
- Дополнительные уровни/биомы/акты.
- Расширенная боевая система (комбо, несколько видов оружия, навыки).
- Продвинутый AI врагов и разнообразие архетипов.
- Сюжетные катсцены и глубокий narrative layer.
- Мета-прогрессия (инвентарь, прокачка, внутриигровая экономика).
- Мультиплеер и социальные функции.
- Полноценная облачная синхронизация сохранений/аккаунтов.
- Локализация на несколько языков и расширенные accessibility-фичи.

## 5) Метрики успеха MVP
### Продуктовые метрики
- **Demo completion rate** (доля игроков, прошедших демо-уровень): целевое значение >= 35%.
- **D1 retention** (возврат на следующий день): целевое значение >= 20% для раннего тестового трафика.
- **Средняя длительность сессии:** 8-15 минут.
- **Tutorial/onboarding success rate** (доля игроков, завершивших onboarding шаги): >= 80%.
- **Crash-free sessions:** >= 98%.

### Технические метрики
- Web build проходит CI на каждом PR.
- Без блокирующих багов на критическом пути (старт -> бой -> финиш).
- Время загрузки первой сцены на целевом окружении: до 5 секунд при нормальном соединении.

## 6) Ограничения и допущения
- **Ограничение по сроку:** MVP выпускается в рамках одного цикла разработки (без расширения scope).
- **Ограничение по контенту:** только один уровень и минимальный набор ассетов.
- **Ресурсное ограничение:** при задержке финальных ассетов используются плейсхолдеры без блокировки тестов.
- **Платформа:** desktop browser first; мобильная оптимизация не является gate для MVP.
