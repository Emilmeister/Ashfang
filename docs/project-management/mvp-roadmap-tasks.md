# Декомпозиция задач для выпуска MVP Ashfang: Echoes of the Fallen Wild

- Статус: active
- Владелец: @devops-agent
- Последняя проверка: 2026-03-16
- Следующий срок ревизии: 2026-03-30
- Связанные артефакты: [Linear: DEV-7]

## 1. Контекст
Нужно подготовить прозрачный набор задач, который доведет проект от текущего состояния до выпуска игрового MVP в браузере на ПК.

## 2. Цель
Сформировать приоритизированный backlog MVP с этапами, зависимостями и критериями готовности.

## 3. Основное содержание

## Этап 0. Инициация и подготовка окружения (Sprint 0)

### TASK-01 — Утвердить scope MVP и критерии успеха
- **Приоритет:** P0
- **Результат:** краткий one-pager со списком фич MVP/Out of scope.
- **Критерии готовности:**
  - зафиксированы 3–5 ключевых сценариев игрока;
  - определены метрики MVP (например, retention D1, длительность сессии, % прохождения демо-уровня);
  - согласованы ограничения по контенту и времени разработки.

### TASK-02 — Создать репозиторий игрового клиента на TypeScript + Phaser 3 + Vite
- **Приоритет:** P0
- **Результат:** стартовый проект с базовой структурой директорий.
- **Критерии готовности:**
  - проект запускается локально (`npm run dev`);
  - настроены ESLint + Prettier + базовый tsconfig;
  - добавлен шаблон сцен `Boot`, `Menu`, `Level`.

### TASK-03 — Настроить CI pipeline для quality gate
- **Приоритет:** P0
- **Результат:** GitHub Actions/GitLab CI для lint/test/build.
- **Критерии готовности:**
  - pipeline запускается на каждый PR;
  - есть обязательная проверка сборки;
  - фиксация артефакта сборки (web bundle).

## Этап 1. Игровой вертикальный срез (Vertical Slice)

### TASK-04 — Реализовать управление персонажем и базовую физику
- **Приоритет:** P0
- **Результат:** перемещение, коллизии, ограничения по карте.
- **Критерии готовности:**
  - WASD/стрелки;
  - корректные коллизии с миром;
  - стабильные 60 FPS на целевом уровне с базовым контентом.

### TASK-05 — Добавить боевую механику MVP
- **Приоритет:** P0
- **Результат:** одна атака игрока + один тип врага.
- **Критерии готовности:**
  - урон, HP, смерть врага;
  - тайминги/кулдауны работают корректно;
  - есть визуальный и/или звуковой фидбек попадания.

### TASK-06 — Собрать первый играбельный уровень
- **Приоритет:** P0
- **Результат:** один завершенный уровень на 5–10 минут.
- **Критерии готовности:**
  - старт/финиш и условия победы;
  - размещены враги/препятствия;
  - есть базовый баланс сложности.

### TASK-07 — Внедрить HUD и меню паузы
- **Приоритет:** P1
- **Результат:** отображение HP, статуса и паузы.
- **Критерии готовности:**
  - HUD обновляется в реальном времени;
  - пауза останавливает геймплей;
  - возврат в игру и в меню работает без ошибок.

## Этап 2. Контент и UX полировка MVP

### TASK-08 — Интегрировать базовые ассеты (графика, анимации, звук)
- **Приоритет:** P1
- **Результат:** единый art/audio пакет для MVP-уровня.
- **Критерии готовности:**
  - ассеты оптимизированы для web;
  - нет блокирующих ошибок загрузки;
  - у ключевых действий есть аудио-фидбек.

### TASK-09 — Добавить onboarding внутри уровня
- **Приоритет:** P1
- **Результат:** подсказки управления и целей.
- **Критерии готовности:**
  - игрок без внешних инструкций понимает базовый loop;
  - подсказки не блокируют управление;
  - сценарий можно пройти с первого запуска.

### TASK-10 — Реализовать сохранение прогресса MVP
- **Приоритет:** P2
- **Результат:** local save (например, localStorage).
- **Критерии готовности:**
  - сохраняются минимум настройки + прогресс прохождения;
  - после перезагрузки данные восстанавливаются;
  - предусмотрен reset save.

## Этап 3. Качество, наблюдаемость и релиз

### TASK-11 — Покрыть критический путь тестами
- **Приоритет:** P0
- **Результат:** unit + e2e smoke.
- **Критерии готовности:**
  - unit-тесты для ключевой логики (урон, состояние игрока, победа/поражение);
  - e2e smoke на старт игры и прохождение основного сценария;
  - тесты запускаются в CI.

### TASK-12 — Подключить аналитику MVP
- **Приоритет:** P1
- **Результат:** события для продуктовых решений.
- **Критерии готовности:**
  - трекаются: старт сессии, смерть, победа, выход;
  - события документированы;
  - данные доступны команде для анализа.

### TASK-13 — Подготовить deployment и release checklist
- **Приоритет:** P0
- **Результат:** staging + production deployment.
- **Критерии готовности:**
  - есть автоматизированный деплой;
  - описан rollback;
  - релизный чек-лист заполнен и пройден.

### TASK-14 — Провести MVP playtest и оформить feedback backlog
- **Приоритет:** P0
- **Результат:** отчет по тестированию на реальных пользователях.
- **Критерии готовности:**
  - минимум 5–10 playtest-сессий;
  - собраны количественные и качественные наблюдения;
  - сформирован список улучшений post-MVP.

## 4. Ограничения и риски
- Ограниченный объем контента: только один завершенный уровень для MVP.
- Риск задержек по арт-ассетам: использовать временные заглушки до финального контента.
- Риск расползания scope: любое расширение после TASK-06 проходит отдельное согласование.

## 5. Критерии готовности/актуальности
- Backlog покрывает путь от инициации до релиза MVP.
- Для каждой задачи определены приоритет и критерии готовности.
- Документ ревизуется не реже одного раза в 2 недели.
