## 1. Цель

Задать единые инженерные правила для кода, чтобы MVP развивался предсказуемо и оставался удобным для изменений.

## 2. Базовые правила

1. **Один PR = одна цель**
   - Не смешивать крупный рефакторинг и новую gameplay-фичу.

2. **Сцены и логика**
   - `src/scenes/*` содержат orchestration и жизненный цикл сцены.
   - Большая внутренняя логика сцены выносится в подпапки (`src/scenes/<scene-name>/*`) в контроллеры/модули.

3. **Ограничение на размер файла**
   - Любой production-файл должен быть **меньше 300 строк кода**.
   - Если файл приближается к лимиту (250+ строк), нужно заранее выносить:
     - константы;
     - типы;
     - подсистемы UI;
     - боевую/физическую логику;
     - narrative/flow-логику.

4. **Явные контракты между модулями**
   - Для взаимодействия модулей использовать typed-структуры и callback API.
   - Не тянуть лишние зависимости между сценами.

5. **Качество перед merge**
   - Обязательно: `npm run typecheck`, `npm run lint`, `npm run build`.

- [ ] Я обновил документацию, если поведение или структура изменились.
