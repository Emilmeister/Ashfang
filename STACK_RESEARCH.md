# Исследование стека для браузерной игры

## Цель
Выбрать технологический стек для разработки игры **Ashfang: Echoes of the Fallen Wild**, которая должна запускаться в браузере на ПК.

## Критерии выбора
- Быстрый старт и удобная разработка (DX).
- Поддержка 2D-графики и анимации.
- Наличие экосистемы (плагины, туториалы, инструменты).
- Простая сборка и деплой в web.
- Возможность масштабирования под более сложную механику.

## Рассмотренные варианты

### 1) TypeScript + Phaser 3 + Vite (кандидат)
**Плюсы:**
- Phaser — зрелый 2D-движок для браузера.
- TypeScript упрощает поддержку и развитие кода.
- Vite дает быстрый dev-сервер и сборку.
- Низкий порог входа для web-команды.

**Минусы:**
- Фокус на 2D; для 3D придется менять стек или подключать отдельные решения.

### 2) TypeScript + PixiJS + собственная игровая логика
**Плюсы:**
- Высокая гибкость рендера.
- Можно построить минималистичную архитектуру под проект.

**Минусы:**
- Нужно самостоятельно реализовывать то, что в Phaser есть из коробки (сцены, коллизии, базовая game-loop инфраструктура).
- Выше стоимость разработки в начале.

### 3) Unity WebGL
**Плюсы:**
- Мощный редактор и готовые инструменты.
- Быстрый прототипинг для сложных механик.

**Минусы:**
- Тяжелые билды для web, дольше загрузка.
- Сложнее интеграция в веб-инфраструктуру и CI.
- Для небольшой браузерной 2D-игры часто избыточно.

### 4) Godot (HTML5 export)
**Плюсы:**
- Open source, удобен для 2D.
- Хорошие инструменты для игрового workflow.

**Минусы:**
- HTML5-экспорт может иметь ограничения/нюансы производительности в зависимости от сцены и ассетов.
- Меньше web-ориентированной экосистемы, чем у JS-стека.

## Рекомендация
Для текущей задачи оптимально выбрать:

## **TypeScript + Phaser 3 + Vite**

### Почему именно этот стек
1. Лучший баланс между скоростью разработки и техрисками для браузерной 2D-игры.
2. Простая публикация как статического web-приложения.
3. Команда сможет быстрее итерировать по геймплею и контенту.
4. Легко интегрировать аналитики, авторизацию, CDN и стандартные web-пайплайны.

## Предлагаемый базовый стек проекта
- **Язык:** TypeScript
- **Движок:** Phaser 3
- **Сборка:** Vite
- **UI (поверх canvas, при необходимости):** React (опционально)
- **Состояние/UI-данные:** Zustand (опционально)
- **Линтинг/форматирование:** ESLint + Prettier
- **Тесты:** Vitest (unit), Playwright (e2e smoke)
- **CI:** GitHub Actions (lint + test + build)
- **Деплой:** Vercel / Netlify / S3 + CloudFront

## Архитектурный каркас (MVP)
- `src/game/scenes` — игровые сцены (Boot, Menu, Level)
- `src/game/entities` — игрок, враги, объекты
- `src/game/systems` — бой, инвентарь, квестовая логика
- `src/ui` — DOM/UI слой (HUD, меню)
- `src/shared` — константы, конфиг, утилиты

## Риски и меры
- **Риск:** разрастание логики в сценах Phaser.
  - **Мера:** early-разделение на systems/entities + код-ревью по архитектуре.
- **Риск:** проблемы производительности при большом числе объектов.
  - **Мера:** object pooling, профилирование, ограничение активных сущностей.
- **Риск:** смешение game state и UI state.
  - **Мера:** отдельные слои состояния и строгие интерфейсы между ними.

## Итог
Рекомендуется утвердить стек **TypeScript + Phaser 3 + Vite** как основной для запуска MVP браузерной версии игры на ПК.
